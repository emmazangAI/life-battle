# 兔子 Still Life 图案设计探索日志

**日期：** 2026-05-06  
**目标：** 为"守护小动物"和"召唤小动物"游戏模式设计一个在康威规则下稳定的像素动物图案

---

## 1. 需求背景

在现有元胞自动机游戏基础上，增加两个新模式：
- **守护模式**：棋盘中央预置一个稳定的像素动物图案（Still Life），玩家在周围画细胞，观察演化是否会破坏动物
- **召唤模式**：给出目标动物形状，玩家找到初始布局，经 N 步演化后尽量接近目标

**康威 Still Life 约束：**
- 每个活细胞恰好有 2 或 3 个邻居
- 每个紧邻的死细胞不能恰好有 3 个邻居（否则会繁殖）

---

## 2. 动物选型过程

### 考虑过的动物

| 动物 | 结论 | 原因 |
|------|------|------|
| 🐸 青蛙 | ❌ 放弃 | 中文互联网语境下有政治敏感联想风险 |
| 🐻 小熊 | ❌ 不够像 | block耳+pond脸的组合太分散，辨识度低 |
| 🦉 猫头鹰 | ✅ 备选 | 双竖pond大眼+beehive嘴，稳定且辨识度可以 |
| 🐰 兔子 | ✅ 最终选择 | 长耳朵是最强辨识特征，像素化后仍然容易认出 |

### 为什么选兔子
- 耳朵是强辨识特征，即使像素化也一眼能看出
- 可以用 block/beehive 竖排做出"长耳"效果
- 孩子喜欢，形象友好

---

## 3. 技术探索：哪些结构在康威规则下稳定

### 已知稳定基础结构（Still Life）

| 名称 | 形状 | 大小 |
|------|------|------|
| Block | ■■ / ■■ | 2×2 |
| Beehive | ·■■· / ■··■ / ·■■· | 4×3 |
| Loaf | ·■■· / ■··■ / ·■·■ / ··■· | 4×4 |
| Pond | ·■■· / ■··■ / ■··■ / ·■■· | 4×4 |
| Tub | ·■· / ■·■ / ·■· | 3×3 |
| 竖Beehive | ·■· / ■·■ / ■·■ / ·■· | 3×4 |

### 关键发现

1. **大于 pond (2×2内空) 的环形结构全部不稳定** — 包括 3×3环、octagon2、tubby 等
2. **多个稳定结构只要间距 ≥ 1 行/列，组合后仍然稳定**
3. **block 竖排叠加（间距1行）是最简单的"实心长条"表达方式**
4. **竖 beehive 叠加可以表达"空心长条/轮廓线"**

---

## 4. 兔子设计迭代过程

### 第一轮：空心结构拼接

尝试用 tub/beehive 做耳朵 + pond 做脸：
- ✅ tub耳 + pond脸（稳定，但太抽象）
- ✅ 竖beehive耳×2层 + pond脸（稳定，耳朵像轮廓线）
- ✅ 竖beehive耳×3层 + pond脸（稳定，耳朵很长）
- ❌ 问题：脸太小，跟耳朵比例失调，不像兔子

### 第二轮：尝试做大圆脸

- ❌ 大环(3×3内空) — 不稳定
- ❌ 竖椭圆(4高2宽内空) — 不稳定
- ❌ 水平椭圆 — 不稳定
- ✅ 用多个 pond/block/beehive 围成"视觉上的大脸"

### 第三轮：参考像素兔子图片

收到3张参考图后，分析关键特征：
- 图1：简约头像，实心轮廓耳+圆脸+眼睛点
- 图2：全身粉色兔，长细耳+大脸+身体
- 图3：网格兔，详细耳朵+圆身体

### 第四轮：最终方案确定

结合参考图1的风格，采用：
- **实心 block 竖排** 做耳朵（比空心beehive更像参考图的"实心耳"）
- **beehive + block + beehive** 围成脸（额头+眼/颊+嘴形成闭合轮廓）

- 嘴巴 = 1 × beehive (水平)
