# 游戏模式设计方向（讨论摘要）

**日期：** 2026-05-06  
**状态：** 方案确认中

---

## 项目背景

基于现有的元胞自动机（康威生命游戏）网页项目 `auto-cell`，扩展新的游戏模式，增加趣味性和教育意义，适合小朋友玩。

起因：五一假期陪孩子看《数学漫步之旅》生命游戏一集，孩子对此非常感兴趣。

---

## 新增游戏模式

### 模式 1：守护兔子 🛡️🐰

- 棋盘中央预置一个像素化兔子图案（康威规则下的 Still Life，永远不会自行变化）
- 玩家在周围随意画细胞
- 点击运行，观察周围细胞如何演化
- 目标：兔子能保持完整不被"入侵"破坏
- 计分：兔子存活步数 / 完整度百分比
- 教育意义：孩子会自然好奇"为什么兔子不会变？" → 理解 Still Life 的条件

### 模式 2：召唤兔子 🎯🐰

- 棋盘显示一个"目标兔子"的虚影（半透明轮廓）
- 玩家在棋盘上自由画初始布局
- 运行 N 步后，系统对比当前状态和目标兔子的相似度
- 提供提示系统帮助孩子理解规则
- 难度设计：
  - 简单：只缺少 1-2 个部件，其余预填
  - 中等：缺少更多部件
  - 困难：完全空白

### 模式切换

- 新页面（`auto_cell_rabbit/index.html`）独立于现有 single 和 multi 版本
- 专属配色和 UI 主题（兔子/自然风格）
- 支持切换到多种族 (multi) 模式

---

## 设计原则

1. **规则不变** — 保持标准康威规则，纯粹教育目的
2. **动物图案是真 Still Life** — 不用特殊机制保护，孩子问"为什么不变"时可以用规则解释
3. **渐进难度** — 从观察到操作到逆向挑战
4. **好玩优先** — 不是题目，是玩具

---

## 技术约束

- 纯原生 HTML + Canvas + JS（与现有项目一致）
- 无构建工具依赖
- 移动端/iPad 友好
- 基于现有 `Grid` / `MultiGrid` 类扩展
