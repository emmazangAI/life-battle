# 兔子游戏最终设计方案

**日期：** 2026-05-07  
**版本：** auto_cell_rabbit v1.0  
**设计师：** Emma + DataMind  

---

## 1. 设计思路转变

### 之前的方案（5月6日）
- 目标：设计一个在康威规则下**完全稳定**的 Still Life 兔子
- 约束：严格使用 block/beehive 等稳定结构拼接
- 问题：兔子辨识度低，不够"像兔子"

### 最终方案（5月7日）
- 目标：**初始图案自由绘制，演化后自然变化**
- 理念：
  > "初始兔子不受康威规则限制，只求视觉上像兔子。  
  > 游戏启动后按标准康威规则演化。  
  > 让小朋友观察：精心设计的图案，在简单规则下会变成什么样子。"

## 2. 兔子图案设计

### 参考来源
- 用户提供的像素兔子图片（可爱、卡通风格）

### 设计特征
- **长耳朵** - 兔子的最强辨识特征
- **圆头** - 额头、脸颊、嘴巴形成完整轮廓
- **清晰五官** - 眼睛、鼻子、嘴巴
- **完整身体** - 上身、腿、脚

### 技术规格
- **尺寸：** 约 11×24 格（宽×高）
- **细胞数：** 150+ 个活细胞
- **颜色：** 默认绿色（种族1），支持多种族切换
- **坐标系统：** 相对坐标 [dx, dy]，原点在兔子中心

### 部件构成
```javascript
RABBIT_PATTERN = {
  leftEar: [...],    // 左耳
  rightEar: [...],   // 右耳
  head: [...],       // 头部轮廓
  eyes: [...],       // 眼睛
  face: [...],       // 脸部填充
  nose: [...],       // 鼻子
  mouth: [...],      // 嘴巴
  body: [...],       // 身体
  legs: [...]        // 腿和脚
}
```

## 3. 游戏模式设计

### 模式一：👀 观察模式（主模式）
**目标受众：** 所有年龄段，尤其是小朋友

**玩法流程：**
1. 进入游戏 → 看到可爱的像素兔子
2. 点击"开始演化" → 兔子开始按康威规则变化
3. 观察兔子形状的演化过程

**教育价值：**
- 理解"复杂性来自简单规则"
- 观察形态变化的不可预测性
- 启发对计算机模拟和生命规则的思考

**UI 特点：**
- 简洁明了，一键启动
- 实时显示代数和存活细胞数
- 初始状态会用红色边框高亮兔子区域

### 模式二：🛡️ 守护模式（挑战模式）
**目标受众：** 有一定理解能力的孩子/成人

**玩法流程：**
1. 切换到"守护模式"
2. 在兔子周围点击网格绘制细胞
3. 设计防护布局（试图让兔子稳定或存活更久）
4. 点击"开始演化" → 观察效果

**挑战目标：**
- 让兔子区域的存活细胞数保持 > 30%
- 或者创造尽可能长寿的演化链

**技术要点：**
- 支持多种族模式（可以用不同颜色的细胞）
- 可切换当前绘制的种族
- 实时统计种族分布

## 4. 技术实现

### 核心类：RabbitGrid
继承并扩展多种族网格逻辑，增加：
- `placeRabbit()` - 在中心放置兔子
- `clearAll()` - 清空网格
- `getRabbitCellCount()` - 统计兔子区域存活细胞
- 支持单种族/多种族模式切换

### 文件结构
```
auto_cell_rabbit/
├── index.html           # 主页面（模式切换、设置面板）
├── rabbit-style.css     # 自定义样式（模式按钮、状态显示）
├── rabbit-pattern.js    # 兔子图案定义（坐标数组）
├── rabbit-grid.js       # 网格逻辑（演化规则、绘制）
└── rabbit-main.js       # 主程序（事件监听、UI 更新）
```

### 与现有版本的关系
- **复用：** `src/style.css` 基础样式
- **参考：** `src/cell-multi.js` 多种族逻辑
- **创新：** 兔子图案系统、模式切换、教育性 UI

## 5. UI/UX 设计

### 布局
- **左上角：** 模式切换按钮（观察/守护）
- **右上角：** 设置浮窗按钮
- **左下角：** 游戏状态（代数、兔子存活数）
- **右侧滑出：** 设置面板（控制、速度、多种族）

### 颜色方案
- **主色：** 绿色 (#00cc66) - 生命、自然
- **强调色：** 红色边框（初始兔子高亮）
- **背景：** 白色网格 + 浅灰线条
- **多种族：** 8 种预设颜色可选

### 交互设计
- **观察模式：** 鼠标默认样式，不可点击网格
- **守护模式：** 鼠标十字准星，点击绘制/擦除
- **设置面板：** 右滑出，点击外部自动收起
- **按钮反馈：** Hover 效果、活动状态高亮

## 6. 教育价值

### 核心概念
1. **涌现（Emergence）** - 简单规则产生复杂行为
2. **不可预测性** - 初始状态微小变化导致结果巨变
3. **规则与自由** - 初始设计自由，演化遵循规则

### 适用场景
- 小学科学课（计算机模拟、生命规则）
- 编程入门（像素艺术、算法可视化）
- 亲子互动（观察模式简单，守护模式有挑战）

### 延伸思考
- "如果改变规则，会怎样？"（引导到 Rulesets 探索）
- "能设计出完全稳定的图案吗？"（引导到 Still Life 研究）
- "多个兔子会怎样？"（引导到生态模拟）

## 10. 总结

这个版本的兔子游戏成功平衡了：
- **视觉吸引力** - 可爱的兔子图案
- **科学严谨性** - 标准康威规则
- **教育价值** - 理解涌现和复杂性
- **互动乐趣** - 观察和挑战两种模式

关键突破是**放弃了 Still Life 约束**，允许初始图案自由设计，让兔子更像兔子，同时保留了康威规则的科学内核。

**下一步：** 测试、优化、收集反馈！
